SFRFs Toolbox - Release Notes

Version 1.0.0

This is the first stable release of the SFRFs (Spectral Fault Receptive Fields) Toolbox, providing a complete pipeline for computing biologically-inspired spectral fault representations from vibration signals. The toolbox supports dataset handling, fault frequency modelling, receptive field construction, scalable ensemble processing, and traceable logging across parallel workers.

Key Features

End-to-end SFRF computation pipeline

You can now go from raw vibration signals to SFRF representations in a fully modular and reproducible way:
  • Bearing parameter definition
  • Fault frequency band computation
  • Receptive field (centersurround) creation
  • SFRF evaluation in time or frequency domain
  • Aggregation across full ensembles
Condition Indicator computation:
  • SFRFsCompute

Explicit modelling of bearing fault frequencies

Full support for standard rolling bearing fault modes:
  • Outer race (BPFO)
  • Inner race (BPFI)
  • Ball (BSF)
  • Cage (FTF)
Implemented via:
  • FaultFrequencyBands
  • BearingFrequencyBands
  • ParametersRollingBearings
  • OperatingConditions
Supports multiple speeds and loads in a single object.

Receptive Field Gain Functions (RFGFs)

Biologically inspired frequency-domain filters, including:
  • Center and surround bands
  • Gaussian and super-Gaussian masks
  • Per-fault and per-condition definition
Implemented via:
  • FrequencyMask
  • ReceptiveFieldGainFunctions
  • SFRFsParameters
  • SFRFsParametersRollingBearings

Robust ensemble processing

Designed to work naturally with fileEnsembleDatastore:
  • Consistent column naming and mapping
  • Automatic suffix handling
  • Safe registry and reconfiguration
  • Vectorized multi-signal operation
Core components:
  • EnsembleBroker
  • SFRFsEnsembleBroker
  • EnsembleProcessor
  • FFTEnsembleProcessor
  • SFRFsEnsembleProcessor
  • EnsembleDatastoreRegistry
  • EnsembleUtil

Dataset integration (XJTU-SY)

Built-in support for the XJTU-SY rolling bearing dataset and extensibility to other datasets:
  • One-call creation and registration
  • Automatic ensemble configuration
  • Registry-based retrieval
Functions:
  • createXJTUSYEnsemble
  • getXJTUSYEnsemble

Per-worker logging and diagnostics

A robust logging system designed for parallel execution:
  • One log file per MATLAB worker
  • Configurable via logging.properties
  • Optional call-site metadata (file, method, line)
  • Helper for interactive inspection (showlog)
Class:
  • SFRFsLogger

Technical Highlights

  • Supports both time-domain (FFT inside) and precomputed frequency-domain input
  • Handles multi-signal matrices natively (column-wise processing)
  • Enforces strict parameter validation
  • Maintains full traceability via tables (conditions, fault groups, masks, outputs)
  • Batch-safe: safe to re-run without corrupting ensemble structure
  • Suitable for:
  • Fault detection
  • Feature extraction
  • Condition monitoring
  • Prognostics pipelines

Validation

The core algorithms are verified against a legacy Difference-of-Gaussians (DoG) reference implementation.
Included test classes:
  • TestFaultFrequencyBands
  • TestBearingFrequencyBands
  • TestReceptiveFieldGainFunctions
  • TestSFRFsCompute
  • Plus supporting unit tests for ensemble and registry logic
All tests validate numeric equivalence within tight tolerances.
Additional considerations
  • SimpleFormatter for Java logging (must be configured before MATLAB starts using _JAVA_OPTIONS).
  • Fault type is explicit by design (via abstract classes).

Next planned extensions

  • Integration with dashboard / demonstrator
  • Expanded dataset adapters
  • Time and angular domain receptive fields
  • Parameter optimization for diagnosis and prognostic pipelines
